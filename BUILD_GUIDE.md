# Руководство по сборке AURION OS (.exe)

Этот проект настроен для сборки в автономный исполняемый файл с использованием `pkg` и `esbuild`.

## 📂 Структура сборки

После запуска команды сборки, в папке `build/` появятся следующие файлы:

*   `react-example-win.exe` — Исполняемый файл для Windows.
*   `react-example-linux` — Исполняемый файл для Linux.
*   `better_sqlite3.node` — **ВАЖНО:** Это нативный модуль базы данных.
*   `README_IMPORTANT.txt` — Инструкция по замене нативного модуля.

## 🚀 Как собрать (на вашем компьютере)

1.  Установите **Node.js** (версии 18+).
2.  Откройте терминал в папке проекта.
3.  Установите зависимости:
    ```bash
    npm install
    ```
4.  Запустите сборку:
    ```bash
    npm run build:exe
    ```

## ⚠️ ВАЖНО: Нативные модули (Windows)

Файл `better_sqlite3.node`, который создается при сборке на Linux (или в облаке), **НЕ БУДЕТ РАБОТАТЬ НА WINDOWS**.

**Что нужно сделать, если вы запускаете .exe на Windows:**

1.  Установите `better-sqlite3` на Windows машине:
    ```bash
    npm install better-sqlite3
    ```
2.  Найдите файл `better_sqlite3.node` в папке `node_modules/better-sqlite3/build/Release/`.
3.  Скопируйте этот файл в ту же папку, где лежит ваш `.exe` файл, заменив существующий.

Без этого шага приложение вылетит с ошибкой "Module not found" при запуске.

## 🛠 Устранение неполадок

*   **Ошибка "pkg not found"**: Убедитесь, что вы запускаете `npm run build:exe`, а не просто скрипт.
*   **Ошибка "better_sqlite3.node"**: См. пункт выше. Нативные модули должны соответствовать операционной системе.
*   **Антивирус**: Некоторые антивирусы могут ложно срабатывать на файлы, упакованные через `pkg`. Это известная особенность упаковщика.

## 📦 Технические детали

*   **Frontend**: Собирается через Vite в папку `dist`.
*   **Backend**: Собирается через esbuild в один файл `dist-server/index.js` (формат CJS).
*   **Упаковка**: `pkg` берет `dist-server/index.js`, добавляет Node.js runtime и ассеты (frontend), создавая единый файл.
